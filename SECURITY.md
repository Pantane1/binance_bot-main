# Security Policy

## Supported Versions

This project is currently under active development. Security updates will be applied to the latest version.

| Version | Supported |
|--------|-----------|
| Latest | Yes |
| Older versions | No |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

Do not open a public GitHub issue for security vulnerabilities.

Instead contact the maintainer directly:

Email: pantane254@gmail.com

Please include the following information:

• Description of the vulnerability  
• Steps to reproduce the issue  
• Possible impact  
• Suggested fix (if available)

You will receive a response as soon as possible.

---

## Security Best Practices

Users of this software should follow these security practices:

- Never commit API keys to the repository
- Use environment variables for secrets
- Restrict API permissions on Binance
- Enable IP restrictions for API keys when possible

---

## Disclaimer

This software interacts with financial systems and cryptocurrency exchanges.  
Use it at your own risk.
